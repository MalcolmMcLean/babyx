Baby X Colours

Baby X uses 32 bit rgba colours throughout. The basic graphics functions support alpha blending.

Baby X colour basics

A colour is passed about as a BBX_RGBA. This is typedefed to an unsigned long.

Use these macros for creating and accessing the BBX_RGBA type.

#define bbx_rgba(r,g,b,a) ((BBX_RGBA) ( ((r) << 24) | ((g) << 16) | ((b) << 8) | (a) )) #define bbx_rgb(r, g, b) bbx_rgba(r,g,b, 255) #define bbx_red(col) ((col >> 24) & 0xFF) #define bbx_green(col) ((col >> 16) & 0xFF) #define bbx_blue(col) ((col >> 8) & 0xFF) #define bbx_alpha(col) (col & 0xFF)

Note that a Baby X rgba buffer is an array of unsigned chars, with the channels in the order r, g, b, a. So an rgba buffer cannot necessarily be cast to an array of BBX_RGBAs.

Baby X name colours

Baby X also contains a massive list of named colours. To obtain a named colour, call

BBX_RGBA bbx_color(char *str);

There is a huge list of allowed colour names. The colour is looked up by binary search so should be reasonably fast, but don't call for every pixel.
